# Bilingua Quest — Build Plan

*July 2026. Companion to [STRATEGY.md](STRATEGY.md) and [DESIGN-MAP-MODE.md](DESIGN-MAP-MODE.md). This document contains the refined brief ("the improved prompt"), the end-to-end build order, and the open unknowns.*

---

## The improved prompt

> **Build "Bilingua Quest": a cozy SNES-style RPG that is a real map of Fredericton** — downtown core and the northside Main Street strip, true to real geography (sourced from TomTom), with real venues as game locations. Players (18+) create a character and walk the city like a Final Fantasy II/III town: talking to townsfolk, entering cafés, pubs, the market, the library, the Playhouse — and practicing French through short, structured, phrase-choice conversations where **every choice succeeds charmingly**, because the game's one non-negotiable design law is: *no outcome may embarrass a nervous learner.*
>
> The game is the **funnel to real life**: its venues are real Fredericton businesses; its event board carries real French-language happenings (conversation nights, concerts, la Fête nationale de l'Acadie, festival seasons); its endgame is walking into the real Dolan's on a real Thursday for a real moitié-moitié conversation — safely, in public, with the trust ladder from DESIGN-MAP-MODE.md governing all human contact. Francophones are not content — they are **Hôtes**, a starred, capped, celebrated in-game status. The design brief draws on Sutherland's *Alchemy* (permission structures, signalling, psycho-logic), *Made to Stick* (concrete, story-shaped messaging), *Contagious* (STEPPS), classic game design (Nintendo-school onboarding, appointment mechanics, collection loops), and SLA pedagogy (comprehensible input, willingness-to-communicate, task-based learning, affective filter).
>
> Fable orchestrates; swarms of cheaper agents (Sonnet) write content and verify. Build order optimizes for a playable, joyful v0 immediately, a real backend v1 next, and NB-wide expansion mechanics ("when Moncton unlocks…") designed in from day one. The ambition line: **Pokémon GO for language — the game people visit New Brunswick to play.**

## Why a game is the right alchemy (one paragraph)

Every principle from the strategy survives translation into play, and most get stronger. The avatar is the L-plate (you can't blush in pixel art). The phrase palette is the script that makes silence impossible. "Every choice succeeds" is variance-reduction made into physics — the worst case isn't survivable, it's *deleted from the rules*. Play reframes error itself: in a game, a wrong attempt is a turn, not a verdict — which is precisely the affective-filter condition (low anxiety, high input) that language acquisition research keeps prescribing. And fun is the only marketing NB can't buy elsewhere: "there's an SNES game where you walk around actual Fredericton practicing French, and the coffee is real" is a *Contagious*-grade story with the city itself as the trigger.

## What's being built now (v0 — this session)

**`game/index.html`** — a single-file, dependency-free canvas game, playable in any browser and publishable as an artifact:

| System | v0 implementation |
|---|---|
| Boot flow | Title → 18+ gate + name (demo sign-in; real auth is v1) → character creation (6 pixel heroes) → city |
| The city | Tile-based walkable map generated from real TomTom geometry (`game/data/fredericton-pois.json`): Queen/King/Brunswick + cross streets, the river, **both bridges** (Westmorland + the green Bill Thorpe walking bridge), northside Main Street. 18 real venues placed in true relative positions |
| Encounters | Enter a venue → interior scene → 3-turn phrase-choice conversation (FR with EN subtitles), warmth-scored, all outcomes warm; survival phrase (« Pouvez-vous répéter plus lentement ? ») always present |
| Economy | Confidence meter (the only "XP"), venue stamps in a **Passeport**, Hôte NPCs (gold star), per-venue quests |
| Events | In-venue **Rendez-vous board** with sample data: conversation nights, Fête de l'Acadie, market hours (v1 makes these real listings) |
| Safety | 18+ gate; safety card in the Passeport (public places, tell a friend, trust your instincts — the 2025 build's rules); no chat of any kind in v0 — human matchmaking is v1+, governed by DESIGN-MAP-MODE's trust ladder |
| Feel | 16px tiles, 2-frame animations, typewriter dialogue boxes, D-pad touch controls, EN/FR UI toggle, optional chiptune bleeps, localStorage saves |

**Content** is generated by a Sonnet swarm (18 venue packs → French-quality review pass → townsfolk + events), orchestrated via Workflow; Fable integrates and verifies with headless-browser playtests.

## Build order (end-to-end)

**v0 — Playable joy (now).** Everything above. Purpose: feel the thing, show the thing, playtest the tone. Zero infrastructure, zero risk.

**v0.5 — Playtest loop (weeks).** Ship v0 link to ~20 real people (immersion grads, francophone friends, one café owner). Watch: do they finish three encounters? Which venue do they enter first? Does the event board make them ask "is this real?" Iterate content and difficulty. Add sound design, seasons (FROSTival snow in winter), more venue interiors.

**v1 — Real accounts, real events, real Hôtes (1–2 months of evenings).** Supabase (or equivalent): auth with real 18+ attestation, profiles + the 2025 quiz (confidence-first), the event board backed by a real calendar (partner: venue owners post their own conversation nights), Hôte applications (capped seats, gold star = vetted human), and the **first real-world loop**: RSVP to a group conversation night at a real venue from inside the game. Group events first — 1:1 matching waits for the trust ladder infrastructure (L1 palettes → L2 → L3) from DESIGN-MAP-MODE.md.

**v1.5 — The trust ladder goes live.** In-game structured exchanges with real humans (palette-only, per DESIGN-MAP-MODE safety model), N=3 progression, then the meetup bridge: the game schedules real moitié-moitiés at partner venues. Moderation tooling. This is the point where the grant conversation (NBBSP, Canadian Heritage Appreciation & Rapprochement — see STRATEGY.md) needs a nonprofit vehicle.

**v2 — New Brunswick expansion.** The map mechanic scales by design: Moncton and Saint John exist on the world map as locked regions with the 50/50 EN/FR unlock bars from the 2025 build. "When Moncton unlocks" is the growth campaign. Tourism angle: a printed *Bilingua Quest passport* at the real venues (stamp your real coffee), Destination-NB pitch — come play the game in person.

## The unknowns Kurt isn't thinking about (asked for explicitly)

1. **Real-venue permission.** Using real business names in a free civic game is likely welcome (it's free advertising) but needs a consent pass before any public launch — one email per venue, an opt-out honoured immediately. (v0 is a private prototype; fine.)
2. **TomTom licensing.** POI data via the connector is fine for prototyping; a public product needs a license check (or a swap to OpenStreetMap, which is trivially licensable and arguably better for this).
3. **Music/IP hygiene.** "SNES-style" is a genre; any actual Nintendo assets/fonts/tunes are not. Everything in v0 is procedurally drawn/synthesized — keep it that way.
4. **Liability & insurance for real meetups** (v1.5+): host-org insurance, venue agreements, incident protocol. Same answer as STRATEGY.md: a fiscal host (CPF NB / library service) carries this better than a sole proprietor.
5. **Moderation duty of care** the moment any human-to-human channel opens — budget a real on-call human, not just tooling. The palette-only design keeps this small but not zero.
6. **Privacy** (PIPEDA): v1 accounts mean a privacy policy, data map, deletion flows — the 2025 build's consent UX is reusable; the obligations are real.
7. **Accessibility**: an action-RPG interface excludes some players the 2025 web app served well; keep a non-game path to every real-world event listing (plain HTML event page behind the same data).
8. **The francophone side must be fun, not dutiful** — playtest with francophones *first*; if Hôte status doesn't feel like an honour to actual francophones, the economy fails regardless of anglophone demand.
9. **Winter.** Half the calendar is -15°C; the event board and venue partnerships need an indoor-season strategy (library, pubs, FROSTival tie-ins), not just patio-season romance.
10. **Burnout math**: one person cannot run live events + moderation + development. The v1 gate is a co-organizer, not a feature.
