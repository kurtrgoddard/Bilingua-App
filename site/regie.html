🧰 SQL d'opération / the ops crib — copy, replace TABLE_ID, run

Le quotidien d'une table, en cinq requêtes. bq-tables décrémente les places tout seul à chaque réservation — le SQL « ajuster » sert aux corrections à la main (désistement, erreur de saisie). A table's daily life in five queries. bq-tables decrements seats on each RSVP by itself; the adjust SQL is for hand corrections (a cancellation, a typo).

-- Marquer complet / mark full (the board shows the gold « complet » pill)
update bilingua.tables set statut = 'complet' where id = 'TABLE_ID';

-- Rouvrir / reopen
update bilingua.tables set statut = 'ouverte' where id = 'TABLE_ID';

-- Annuler / cancel (disparaît du tableau / drops off the board)
update bilingua.tables set statut = 'annulee' where id = 'TABLE_ID';

-- Marquer passée, après la soirée / mark past, after the evening
update bilingua.tables set statut = 'passee' where id = 'TABLE_ID';

-- Lister les réservations d'une table / list the RSVPs for one table
select name, email, parcours, invite_code, created_at
from bilingua.rsvps
where table_id = 'TABLE_ID'
order by created_at;

-- Enlever une place à la main / hand-decrement one seat
update bilingua.tables
set places_restantes = greatest(places_restantes - 1, 0)
where id = 'TABLE_ID';

-- Rendre une place (désistement) / give a seat back (a cancellation)
update bilingua.tables
set places_restantes = least(places_restantes + 1, places)
where id = 'TABLE_ID';

-- Ajuster à un chiffre exact / set the exact remaining count
update bilingua.tables set places_restantes = 3 where id = 'TABLE_ID';

📊 Ce qui se mesure honnêtement / what can be measured honestly

Les seuls compteurs réels sont ceux des formulaires Netlify — et seulement pour les envois faits sur le miroir netlify.app. The only real counters are the Netlify form dashboards, and only for submissions made on the netlify.app mirror.

  • Formulaire « waitlist »Nombre d'inscriptions à la liste d'attente. Waitlist sign-up count.
  • Formulaire « feedback »Nombre de retours envoyés. Feedback submission count.
  • Formulaire « proposer-quete »Nombre de quêtes proposées. Proposed-quest count.
  • Formulaire « proposer-rdv »Nombre de rendez-vous proposés. Proposed-meetup count.

Le champ parcours (facultatif) présent sur les quatre formulaires donne la répartition francophones vs apprenant·es — avec le consentement de la personne, jamais autrement. C'est la seule mesure du pont francophone ↔ apprenant·e, et elle reste volontaire. The optional « parcours » field on all four forms gives the francophone-vs-learner split — with consent only, never otherwise.

Le Rythme / compressed operator checklist

  1. Chaque jour (2 min) — un œil sur la boîte courriel (formulaires + réponses), un œil sur la mission du jour ci-dessus. Daily glance: inbox + today's mission.
  2. Jeudi 17 h 30 — arriver à Dolan's, cartes d'invitation en poche (« Les Codes »), étoile dorée visible. Arrive early with printed codes.
  3. Jeudi 18 h – 20 h — le Rendez-vous : accueillir, présenter, laisser les Serments du Pont se prêter (« Dépannage » si un code bloque). Host the rendez-vous; troubleshoot oaths.
  4. Jeudi soir — noter la présence réelle dans le carnet. Des chiffres honnêtes, rien d'autre. Record honest attendance numbers.
  5. Vendredi — canoniser la meilleure quête proposée (« Canonisation »), remercier l'auteur par courriel. Canonise one quest, thank its author.
  6. Lundi — vérifier Netlify Forms et que le dernier déploiement GitHub Actions est vert. Check forms + green deploys.
  7. Chaque mois, 20 h 02 — le rituel mensuel (réservé — à définir). Monthly 20h02 placeholder.
  8. Fin de saison — exporter le CSV des codes imprimés, photographier les cartes restantes, écrire la rétro. Season close: CSV export, photo of leftover cards, retro notes.
  9. Le grand cahier de régie complet : OPERATIONS.md (à la racine du dépôt). Full runbook, at the repo root.
